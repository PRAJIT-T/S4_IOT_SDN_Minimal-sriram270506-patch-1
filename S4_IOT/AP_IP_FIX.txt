def get_actual_ip(self):
    """Get the actual IP address of this machine"""
    try:
        # Connect to controller to determine our IP
        s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
        s.connect((self.controller_ip, self.controller_port))
        ip = s.getsockname()[0]
        s.close()
        return ip
    except:
        # Fallback: try to get from hostname
        try:
            return socket.gethostbyname(socket.gethostname())
        except:
            return "127.0.0.1"

How it works:
  • Connects to controller to determine routing
  • getsockname()[0] gives the IP that would be used for that connection
  • Fallback to hostname resolution
  • Final fallback to localhost (only in demo mode)

TECHNICAL DETAILS
══════════════════

Python Method: getsockname()
───────────────────────────

s = socket.socket(socket.AF_INET, socket.SOCK_DGRAM)
s.connect(('192.168.1.100', 9000))
local_ip = s.getsockname()[0]
s.close()

What this does:
  • Creates a UDP socket
  • Connects it to controller
  • OS determines which interface to use
  • getsockname() returns that interface's IP
  • This is the ACTUAL IP that would be used
  • Not just a guess, but kernel-determined

Why better than other methods:
  • Doesn't rely on hostname resolution
  • Doesn't depend on network configuration
  • Returns the IP the OS actually uses
  • Works with complex routing


FALLBACK CHAIN
───────────────

1st Choice: getsockname() via controller connection ✓ Best
   └─ Returns actual interface IP
   └─ Most reliable

2nd Choice: socket.gethostbyname(socket.gethostname()) ✓ Good
   └─ Resolves local hostname to IP
   └─ Works on most systems

3rd Choice: Return "127.0.0.1" ✓ Fallback
   └─ Only if both above fail
   └─ Better than nothing
