COMPONENT DESCRIPTIONS
======================

1. CONTROLLER (sdn_controller_multi_laptop.py)
   ────────────────────────────────────────
   Role: Central decision maker
   Runs on: Laptop 1 (192.168.1.101)
   
   What it does:
   • Listens on port 9000 for metrics from agents
   • Analyzes metrics every 5 seconds
   • Prints formatted status every 5 seconds
   • Detects jamming attacks using 3-metric threshold:
     - Throughput < 2.0 Mbps
     - RSSI < -60 dBm
     - Packet rate > 5000 pps
   • Sends isolation command to AP when attack detected
   • Tracks metrics by channel
   
   Command to start:
   $ python3 sdn_controller_multi_laptop.py --host 0.0.0.0 --port 9000
   
   Expected output:
   Listening on 0.0.0.0:9000
   Waiting for agent connections...
   [Then after agents connect - metrics every 5 seconds]


2. AP AGENT (ap_agent_multi_laptop.py)
   ─────────────────────────────────
   Role: WiFi Access Point simulator
   Runs on: Laptop 2 (192.168.1.102)
   
   What it does:
   • Simulates 2 virtual WiFi nodes
   • Generates realistic WiFi metrics:
     - Throughput (normal: 4.5 Mbps, attack: 0.9 Mbps)
     - RSSI signal strength (normal: -45 dBm, attack: -70 dBm)
     - Packet rate (normal: 300 pps, attack: 8000+ pps)
   • Sends metrics to controller every second
   • Receives commands from controller
   • Can isolate (block) specific IP addresses
   
   Command to start:
   $ python3 ap_agent_multi_laptop.py --controller 192.168.1.101 --listen 0.0.0.0 --port 9001
   
   Expected output:
   Listening on 0.0.0.0:9001
   Controller at 192.168.1.101:9000


3. MONITOR AGENT (monitor_agent_multi_laptop.py)
   ────────────────────────────────────────────
   Role: Traffic monitor
   Runs on: Laptop 3 (192.168.1.103)
   
   What it does:
   • Generates additional traffic metrics
   • Tracks packet loss and latency
   • Sends metrics to controller every second
   • Helps controller make more accurate decisions
   
   Command to start:
   $ python3 monitor_agent_multi_laptop.py --controller 192.168.1.101
   
   Expected output:
   Monitor Agent Started
   Reporting to controller at 192.168.1.101:9000


4. ATTACKER (attacker_multi_laptop.py)
   ──────────────────────────────────
   Role: Jamming attack simulator
   Runs on: Laptop 4 (192.168.1.104)
   
   What it does:
   • Generates jamming attack packets
   • Sends 100+ jam packets per second to AP
   • Simulates real WiFi jamming behavior
   • Listens for isolation notice from controller
   • Gracefully shuts down when isolated
   
   Command to start:
   $ python3 attacker_multi_laptop.py \
       --target-ap 192.168.1.102 \
       --controller 192.168.1.101 \
       --channel 6 \
       --duration 15
   
   Expected output:
   STARTING JAMMING ATTACK!
   Sent 100 jam packets
   Sent 200 jam packets
   ...
   ISOLATED by controller

STEP-BY-STEP DEPLOYMENT
=======================

PREREQUISITES:
□ 4 laptops with Python 3.7+
□ All on same WiFi network
□ Know IP address of each laptop (use: ifconfig | grep 'inet ')
□ Firewall allows UDP ports 9000-9003
□ Test connectivity: ping between laptops

FILES YOU NEED:

On Laptop 1 (Controller):
  • sdn_controller_multi_laptop.py

On Laptop 2 (AP Agent):
  • ap_agent_multi_laptop.py

On Laptop 3 (Monitor):
  • monitor_agent_multi_laptop.py

On Laptop 4 (Attacker):
  • attacker_multi_laptop.py

Helper files (any laptop):
  • setup_multi_laptop.py
  • MULTI_LAPTOP_DEPLOYMENT_CHECKLIST.txt


QUICK START (15 MINUTES):

1. Get IP addresses on each laptop:
   $ ifconfig | grep 'inet '
   
   Write down:
   Controller IP: 192.168.1.___
   AP IP: 192.168.1.___
   Monitor IP: 192.168.1.___
   Attacker IP: 192.168.1.___

2. Copy files to each laptop

3. LAPTOP 1 - Start Controller:
   $ python3 sdn_controller_multi_laptop.py --host 0.0.0.0 --port 9000
   
   Wait for: "Listening on 0.0.0.0:9000"

4. LAPTOP 2 - Start AP Agent (wait 2 seconds):
   Edit: Change controller_ip = '192.168.1.101' (use your IP)
   $ python3 ap_agent_multi_laptop.py --controller 192.168.1.101
   
   Wait for: "Listening on 0.0.0.0:9001"

5. LAPTOP 3 - Start Monitor (wait 2 seconds):
   Edit: Change self.controller_ip = '192.168.1.101'
   $ python3 monitor_agent_multi_laptop.py --controller 192.168.1.101
   
   Wait for: "Monitor Agent Started"

6. LAPTOP 1 - Watch Controller output:
   Should see metrics printing every 5 seconds showing baseline

7. LAPTOP 4 - Start Attacker (wait 10 seconds):
   $ python3 attacker_multi_laptop.py \
       --target-ap 192.168.1.102 \
       --controller 192.168.1.101 \
       --channel 6 \
       --duration 15

8. WATCH LAPTOP 1 (Controller):
   After 10 seconds, metrics should degrade
   Should see: "JAMMING ATTACK DETECTED!"
   Should see: "ISOLATING ATTACKER!"

COMMON ISSUES & SOLUTIONS
=========================

Issue: "Connection refused" on AP Agent
Solution:
  • Controller must start FIRST
  • Wait 3 seconds before starting AP
  • Check controller IP is correct
  • Verify IP not already in use: netstat -an | grep 9000

Issue: No metrics printing on Controller
Solution:
  • Check AP Agent is running (check its terminal)
  • Verify controller IP in AP script matches
  • Check firewall: sudo ufw allow 9000:9003/udp
  • Ping from AP to controller: ping 192.168.1.101

Issue: Attacker can't connect to AP
Solution:
  • Check AP Agent is running
  • Verify target-ap IP is correct
  • Ensure port 9001 is open
  • Check firewall: sudo ufw allow 9001/udp

Issue: Network latency / slow metrics
Solution:
  • Check WiFi signal strength
  • Move laptops closer together
  • Ensure no interference (microwaves, cordless phones)
  • Switch WiFi channel in attacker command

Issue: Controller doesn't detect attack
Solution:
  • Wait 15+ seconds for metrics to stabilize
  • Check threshold values in controller
  • Verify attacker is running and sending packets
  • Check attacker target-ap IP is correct


CUSTOMIZATION OPTIONS
====================

Change WiFi Channel:
  In sdn_controller_multi_laptop.py:
  self.channel = 11  (instead of 6)
  
  In attacker command:
  --channel 11

Change Metrics Update Interval:
  In sdn_controller_multi_laptop.py:
  Modify: time.sleep(5)  to  time.sleep(2)
  (prints every 2 seconds instead of 5)

Change Attack Duration:
  When running attacker:
  --duration 30  (30 seconds instead of 15)

Change Detection Threshold:
  In sdn_controller_multi_laptop.py:
  Modify the if statement in check_for_attack()
  Example: throughput < 3.0 (instead of 2.0)

Change Attack Intensity:
  In attacker_multi_laptop.py:
  Modify packet generation rate
  time.sleep(0.01)  controls how many packets/sec
