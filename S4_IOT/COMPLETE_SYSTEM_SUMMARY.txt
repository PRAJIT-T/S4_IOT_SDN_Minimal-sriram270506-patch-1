   ────────────────────────────────────
   Purpose: WiFi Access Point simulator with virtual nodes
   Port: 0.0.0.0:9001 (listens for isolation commands)
   Function:
     • Simulates WiFi AP with 2 virtual nodes
     • Generates realistic WiFi metrics
     • Reports metrics to controller
     • Detects isolation commands
   
   Virtual Nodes:
     ✓ Node-1: 192.168.1.10 (MAC: AA:BB:CC:DD:EE:01)
     ✓ Node-2: 192.168.1.11 (MAC: AA:BB:CC:DD:EE:02)
   
   Metrics Generated:
     ✓ Throughput (4.5 Mbps baseline, 0.9 Mbps during attack)
     ✓ RSSI Signal (-45 dBm baseline, -70 dBm during attack)
     ✓ Packet Rate (300-500 pps baseline, 8000+ pps during attack)
     ✓ Packet Loss (0.5% baseline, 75% during attack)
   
   Special Feature: Now detects actual IP address!
     ✓ Calls get_actual_ip() at startup
     ✓ Sends IP in every metrics message
     ✓ Works in single and multi-laptop modes


3. MONITOR AGENT (monitor_agent_multi_laptop.py)
   ───────────────────────────────────────────
   Purpose: Network traffic monitor
   Port: 0.0.0.0:9002 (listens, not used in v1)
   Function:
     • Monitors packet rates in network
     • Reports traffic patterns to controller
     • Sends metrics every second
   
   Metrics:
     ✓ Packet Rate (200-500 pps baseline, 8000+ pps during attack)
     ✓ Packet Loss (0.1-1% baseline, 40-90% during attack)
     ✓ Latency (2-5ms baseline, 50-150ms during attack)


4. ATTACKER (attacker_multi_laptop.py)
   ────────────────────────────────────
   Purpose: Jamming attack simulator
   Port: 0.0.0.0:9003 (receives isolation notice)
   Function:
     • Simulates WiFi jamming attack
     • Sends ~100 pps jam packets
     • Listens for isolation command
     • Stops when isolated
   
   Attack Characteristics:
     ✓ 12-second duration (configurable)
     ✓ 100 packets per second rate
     ✓ Impacts all metrics: throughput↓ RSSI↓ packet_rate↑
     ✓ Triggers attack detection after 5+ packets


5. ORCHESTRATOR (quick_start.py)
   ─────────────────────────────
   Purpose: Automated demo runner
   Function:
     • Starts all components in sequence
     • Manages attack timing
     • Cleans up processes
     • Shows complete workflow

     1. Start controller (wait 3s)
     2. Start AP agent (wait 2s)

     4. Attack for 12 seconds
     5. Show results
     6. Cleanup
