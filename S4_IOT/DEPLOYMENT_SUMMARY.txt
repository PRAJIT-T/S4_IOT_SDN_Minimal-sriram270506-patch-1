🔍 HOW IT WORKS
===============

ATTACK DETECTION:
  • Monitors 3 key metrics from WiFi network
  • Analyzes metrics every 5 seconds
  • Compares against thresholds
  • Triggers detection when ALL 3 exceed limits

THE 3 METRICS:
  1. Throughput must drop below 2.0 Mbps
  2. Signal strength (RSSI) must drop below -60 dBm
  3. Packet rate must exceed 5000 packets/second

RESPONSE:
  • Immediately sends isolation command to AP
  • AP blocks attacker IP address
  • Attacker receives isolation notice
  • System returns to normal operation

🐛 COMMON ISSUES & QUICK FIXES
==============================

"Components can't find each other"
  → Verify all on same WiFi
  → Check IP addresses with: ifconfig | grep 'inet '
  → Test connectivity: ping 192.168.1.102

"No metrics appearing on controller"
  → Check AP agent is running in its terminal
  → Verify controller IP in ap_agent script
  → Check firewall: sudo ufw status

"Attacker won't connect"
  → Ensure controller running on Laptop 1
  → Verify target IP addresses are correct
  → Check firewall allows UDP on ports 9000-9003

"Detection not working"
  → Wait 15+ seconds for system to stabilize
  → Check threshold values in controller
  → Verify attacker is sending packets

For comprehensive troubleshooting:
  → See COMPLETE_MULTI_LAPTOP_GUIDE.txt section "TROUBLESHOOTING"
