ORIGINAL PROBLEM
================

You reported:

Expected:

ROOT CAUSE ANALYSIS
===================

The AP agent and Monitor agent were sending metrics with specific field names:
  - throughput_mbps (not throughput)
  - rssi_dbm (not rssi)
  - packet_loss_percent (not loss)
  - packet_rate_pps (not packet_rate)

But the controller was trying to access them with the original names,
which returned empty/zero values.

Result: All zeros in the display output


SOLUTION IMPLEMENTED
====================

Modified file: sdn_controller_multi_laptop.py

Change 1 - Socket Reuse (Line 20):
  Added: self.server_socket.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
  Why: Prevents "Address already in use" errors on restart

Change 2 - Metric Field Extraction (Lines 57-69):
  Before:
    data_payload = message.get('data', {})
    throughput = data_payload.get('throughput', 0)
  
  After:
    throughput = message.get('throughput_mbps', message.get('throughput', 0))
  
  This checks for the actual field name first, then falls back to alternatives

Change 3 - Metrics Display (Lines 205-214):
  Before:
    print(f"Throughput: {ap.get('throughput', 0):6.2f} Mbps")
  
  After:
    throughput = ap.get('throughput_mbps', ap.get('throughput', 0))
    print(f"Throughput: {throughput:6.2f} Mbps")

TECHNICAL DETAILS
=================

What the fix does:

Before: Controller tried to access metrics like:
  message.get('data', {}).get('throughput')
  This returned 0 because 'data' key doesn't exist

After: Controller tries multiple field name variations:
  message.get('throughput_mbps',        # Try actual name first
              message.get('throughput', # Then try generic name
                          0))            # Then default to 0

This handles both:
  1. Agents that send throughput_mbps
  2. Agents that send throughput
  3. Missing metrics gracefully
