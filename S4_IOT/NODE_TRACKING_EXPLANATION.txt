  MAC Address:  AA:BB:CC:DD:EE:02
  Status:       Connected to AP (unless isolated)

These nodes GENERATE THE METRICS YOU SEE:
  ✓ Throughput (Mbps) - Real packet transmission speed between nodes
  ✓ RSSI (dBm) - Real signal strength at each node
  ✓ Packet Loss (%) - Real packet loss rate experienced by nodes
  ✓ Packet Rate (pps) - Real packet generation rate by nodes


HOW IT WORKS
════════════

1. BASELINE PHASE (Before Attack):
   
   Each virtual node generates realistic WiFi traffic:
   
   Node-1 metrics:
     - Throughput: ~4.5 Mbps (normal WiFi speed)
     - RSSI: -45 dBm (good signal strength)
     - Packet Rate: ~300-400 pps (normal traffic)
     - Packet Loss: ~0.5% (minimal loss)
   
   Node-2 metrics:
     - Same realistic values from connected node
   
   This shows HEALTHY network conditions from real nodes


2. ATTACK PHASE (During Jamming):
   
   Nodes experience degraded metrics:
   
   Node-1 under attack:
     - Throughput: ~0.9 Mbps (drops from 4.5!)
     - RSSI: -70 dBm (signal degrades!)
     - Packet Rate: ~8000-10000 pps (noise increases!)
     - Packet Loss: ~75% (massive loss!)
   
   Controller DETECTS the attack by monitoring these REAL metric changes


3. DETECTION & ISOLATION:
   
   Controller analyzes metrics from nodes and:
   
   ✓ Detects abnormal behavior (throughput↓, RSSI↓, loss↑)
   ✓ Identifies attacking source IP
   ✓ Isolates attacker: blocks IP addresses of attacking nodes
   ✓ Protects legitimate nodes: Node-1 and Node-2 stay online
