DETECTION MECHANISM
═══════════════════

The system detects attacks by monitoring THREE metrics:

1. THROUGHPUT
   Normal: 4.5 Mbps
   Attack: 0.9 Mbps
   Threshold: < 2.0 Mbps = ALERT

2. SIGNAL STRENGTH (RSSI)
   Normal: -45 dBm
   Attack: -70 dBm
   Threshold: < -60 dBm = ALERT

3. PACKET RATE
   Normal: 300 pps
   Attack: 8000+ pps
   Threshold: > 5000 pps = ALERT

Attack is detected when ALL THREE conditions are met simultaneously.

METRICS EXPLAINED
═════════════════

THROUGHPUT (Mbps - Megabits per second)
────────────────────────────────────────
Normal value: 4.5 Mbps
Attack value: 0.9 Mbps
Shows: WiFi data rate

RSSI (dBm - Decibels relative to 1 milliwatt)
──────────────────────────────────────────────
Normal value: -45 dBm (good signal)
Attack value: -70 dBm (weak signal)
Shows: Signal strength (more negative = weaker)

PACKET RATE (pps - Packets per second)
──────────────────────────────────────
Normal value: 300 pps
Attack value: 8000+ pps
Shows: Network traffic intensity

PACKET LOSS (%)
──────────────
Normal value: 0.5%
Attack value: 80%
Shows: Percentage of lost packets

LATENCY (ms - Milliseconds)
──────────────────────────
Normal value: 2-5 ms
Attack value: 50-150 ms
Shows: Network delay

Change Attack Pattern:
  attacker.py: Modify jam_packet generation
