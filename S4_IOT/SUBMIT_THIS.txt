================================================================================
                            TECHNICAL NOTES
================================================================================

APPROACH: Pure Python Simulation of SDN Control Plane
  - Simulates OpenFlow switch behavior without requiring actual hardware
  - Models telemetry collection (similar to Ryu controller + OVS)
  - Proves SDN concepts work in isolation from physical switch constraints
  - Portable: runs on any Linux/macOS/Windows system with Python 3.7+

DETECTION ALGORITHM:
  - Monitors per-AP packet rates every 5 seconds
  - Threshold: 10,000 pkt/s indicates anomalous traffic
  - Identifies source MAC via telemetry
  - Confidence: Jammer consistently detected within 5 seconds of activation

RECOVERY STRATEGY:
  - Controller installs drop rule on detected jammer MAC
  - Rule prevents malicious packets from entering AP queue
  - Alternative strategies available: rate limiting, channel reassignment

METRICS VALIDATION:
  All measurements come from the Monitor class which simulates:
  - Per-AP throughput (Mbps)
  - Per-AP packet rates (pkt/s)
  - Per-AP packet loss (%)
  - Channel assignment
  - Client association
